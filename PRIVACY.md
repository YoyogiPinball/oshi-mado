# プライバシーポリシー — 推し窓 (oshi-mado)

推し窓は、登録した YouTube チャンネルの新着を1画面にまとめて表示する Chrome 拡張機能です。
本ポリシーは、推し窓がどのデータを扱い、どこへ送り、どこに保存するかを説明します。

## 開発者が収集するデータ：なし

推し窓には開発者のサーバーが存在しません。
利用状況の計測（アナリティクス）、クラッシュレポート、広告 ID の取得、第三者へのデータ提供は一切行いません。
あなたが登録したチャンネルや API キーが、開発者の手元に届くことはありません。

## 拡張機能が扱うデータと、その保存場所

| データ | 内容 | 保存場所 |
|-------|------|---------|
| 推しリスト | あなたが登録した YouTube チャンネル ID | `chrome.storage.sync` |
| API キー | あなたが取得した YouTube Data API v3 のキー（任意） | `chrome.storage.sync` |
| 表示設定 | 「動画をアプリで開く」などのトグル | `chrome.storage.sync` |
| 判定キャッシュ | 取得済み動画の種類・時刻の一時保存 | `chrome.storage.local` |

`chrome.storage.sync` は Chrome の同期領域です。あなたが Chrome にログインして同期を有効にしている場合、これらの設定はあなた自身の Google アカウントを通じて、あなたの他の端末の Chrome と同期されます。同期先はあなたのアカウント内に限られ、開発者や第三者はアクセスできません。
`chrome.storage.local` はその端末の中だけに保存され、同期されません。

拡張機能を削除すると、これらのデータも Chrome によって削除されます。

## 外部への通信先

推し窓が通信するのは、次の Google のドメインだけです。

- `https://www.youtube.com/` — チャンネルの公開 RSS フィード（新着の取得）、および `@ハンドル` からチャンネル ID を解決するためのチャンネルページ
- `https://www.googleapis.com/youtube/v3/` — YouTube Data API v3（配信ステータス・開始時刻・チャンネル名とアイコンの取得）

いずれの通信も、あなたが登録したチャンネル ID と、API キーを設定している場合はそのキーを含みます。これらは Google に対して送られ、それ以外の宛先へは送られません。

## API キーについて（BYOD モデル）

推し窓は、開発者が用意した API キーを共有する方式を採っていません。
各ユーザーが Google Cloud Console で自分の API キーを取得し、設定画面から登録して使います（キーの登録は任意で、未登録でも新着一覧は表示されます）。

このキーはあなたの Google Cloud プロジェクトに紐づく認証情報です。次の点にご注意ください。

- キーは他人に渡さないでください。公開リポジトリやスクリーンショットへの貼り付けも避けてください。
- キーの利用によって発生する YouTube Data API のクォータ消費は、あなたのプロジェクトに計上されます。
- 漏洩が疑われる場合は、Google Cloud Console で該当のキーを削除し、作り直して設定し直してください。

## 権限を使う理由

- `storage` — 上表のデータを端末および Chrome の同期領域に保存するため。
- `https://www.youtube.com/*`（ホスト権限） — チャンネルの公開 RSS を取得するため。
