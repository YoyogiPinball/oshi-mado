# 推し窓 (oshi-mado)

推し窓は、登録した YouTube チャンネルの新着動画やライブ配信を1画面に集約する Chrome 拡張機能です。
巡回の手間をなくし、複数チャンネルの新着情報をダイジェスト形式で横断的にチェックできます。

---

## 主な機能

- **新着の横断表示**：登録したチャンネルの直近7日間の投稿を、日付ごとにまとめて表示します。
- **種類別タブとバッジ**：全て / 動画 / アーカイブ / 配信中 / 予定のタブで切り替えられます。カードのサムネイル上には、配信中や開始予定日時を示すバッジが表示されます。
- **視聴可能時刻を基準にしたソート**：ライブ配信やアーカイブは「配信が始まった時刻」、動画や Shorts は「公開された時刻」を基準に並び替えます。
- **段階的なアプローチ**：YouTube Data API キーの登録は任意です。キーが未登録の場合でも、公開 RSS フィードから新着動画のタイトルやURLを読み込んで動作します（APIキーがある場合は、アイコン画像や正確な配信ステータス、開始時刻が補完されます）。
- **タブ表示機能**：ポップアップ画面内のボタンから、ブラウザの通常タブとして開き直すことができます。横幅制限のない広い画面で新着を確認できます。
- **ダークテーマ対応**：ブラウザのテーマ設定（ライト／ダーク）に合わせて、自動で配色が切り替わります。

---

## 仕組みとクォータの節約

本拡張機能は、データ取得の負荷を最小限に抑えるために、公開 RSS と公式 API v3 のハイブリッド構成を採用しています。

1. **新着の発見**：各チャンネルが公開している RSS フィード（`youtube.com/feeds/videos.xml`）を並列で取得します。RSS の取得はクォータ（API利用制限）を消費しません。
2. **情報の補完とキャッシュ**：RSS から得た動画 ID のうち、状態が確定しているもの（通常動画や終了したアーカイブなど）は `chrome.storage.local` にキャッシュします。
3. **API問い合わせの最小化**：キャッシュにないものや、状態が変わりうるもの（配信予定、配信中など）に限り、YouTube Data API v3 の `videos.list` を叩いて情報を取得します。

### クォータ消費の実測値
59のチャンネルを登録し、直近7日間の新着が計400件ある環境での実測値です。
キャッシュと Shorts の自動スキップ機能により、実際に API へ問い合わせた動画は 43 件（配信中 9 件、予定 34 件）に抑えられました。
API 仕様上、1回のリクエストに最大 50 件の動画 ID を含められるため、**1回の更新で消費するクォータはわずか 1 ユニット**でした。
1日あたりの無料枠は 10,000 ユニットであるため、個人利用の範囲で上限に達する心配はありません。

## 技術的な判断の記録

### 非公式エンドポイント `youtubei/v1/player` の不採用
開発初期には、APIキーの登録手順をなくすために内部APIである `youtubei/v1/player` を使用する設計を検証しました。
しかし、Chrome 拡張機能の文脈からこのエンドポイントを叩くと、ブラウザが自動付与する `Origin` ヘッダーが `chrome-extension://...` となり、YouTube 側から 403 Forbidden で接続を遮断されることが判明しました。
これを回避するには `declarativeNetRequest` などを使ってリクエストヘッダーを書き換える（名札を外して要求し直す）必要がありますが、これは明確な意図を持ったアクセスの迂回行為に該当し、YouTube の利用規約（ToS）に抵触する恐れがあります。
そのため、本拡張機能では開発者とユーザーの安全を考慮し、拡張機能からの接続を許可している公式 API v3 を採用し、ユーザー自身で API キーを入力してもらう BYOD モデルを決定しました。

### Google Takeout によるインポートの推奨
登録チャンネルの一括取り込み機能を実装するにあたり、OAuth 認証（`chrome.identity`）を導入すると、Google Cloud 側で認証同意画面のセットアップが必要になり、配布や運用の難易度が跳ね上がります。
そこで、Google 公式のデータ持ち出し機能（Google Takeout）からダウンロードできる `subscriptions.csv` の中身を、設定画面のテキストエリアにそのまま貼り付けて一括追加する設計を採用しました。
これにより、認証のセットアップを不要にしながら、初回のチャンネル一括登録を数ステップで完了できるようにしました。

## 使い方

1. `chrome://extensions` をブラウザで開き、右上の「デベロッパーモード」を ON にします。
2. 「パッケージ化されていない拡張機能を読み込む」を押し、リポジトリの `extension/` フォルダを選択して読み込みます。
3. 拡張機能のポップアップ画面、または設定画面（options.html）を開きます。
4. 設定画面の「推しを追加」欄に、YouTube のチャンネル ID（`UC...`）や、チャンネルの URL、`@ハンドル` を入力して「リストに追加」を押し、最後に「保存」をクリックします。
5. （推奨）「YouTube Data API キー」欄の手順に従って取得した API キーを設定し、「保存して接続テスト」を実行します。

## プライバシーと API キーの扱い

推し窓は、**ユーザー自身が取得した YouTube Data API の認証情報を使う**設計（BYOD モデル）です。開発者が用意したキーを共有する方式ではないため、API の利用枠はあなたの Google Cloud プロジェクトに紐づきます。キーは他人に渡さず、公開リポジトリやスクリーンショットに貼らないでください。

登録したチャンネルと API キーは `chrome.storage.sync`、判定キャッシュは `chrome.storage.local` に保存されます。開発者のサーバーは存在せず、通信先は YouTube と Google API のみです。詳細は [プライバシーポリシー](PRIVACY.md) を参照してください。

## 既知の制約

- **新着の反映遅延**：YouTube に動画がアップロードされてから、RSS や API に反映されるまで数分のタイムラグが発生します。
- **配信予定の取りこぼし**：RSS フィードは直近の 15 件までしか返さないため、公開日がかなり先の配信予定枠などは新着リストに載らない場合があります。
- **開始予定時刻がない配信枠**：一部のライブ配信において、開始予定時刻が設定されておらず「もうすぐ始まります」としか定義されていないケースがあります。その場合は時刻不明として表示します。
- **コミュニティ投稿**：YouTube Data API v3 にコミュニティ投稿を取得する仕組みが提供されていないため、テキストやアンケートの投稿は対象外となります。
