# アイコンアセット

Chrome 拡張機能とストアが使うアイコンです。`manifest.json` の `icons` から参照しています。

| ファイル | サイズ | 使われる場所 |
|---------|-------|-------------|
| `icon16.png` | 16×16 | ツールバー・ファビコン |
| `icon32.png` | 32×32 | Windows の高 DPI 表示 |
| `icon48.png` | 48×48 | 拡張機能の管理画面（`chrome://extensions`） |
| `icon128.png` | 128×128 | インストール時ダイアログ・ストア掲載 |

> **Note:** 拡張機能のアイコンは **PNG のみ**です。`manifest.json` から SVG を指定すると読み込みに失敗します。

## 作り直すとき

デザインの原本は、リポジトリ直下の `oshi_mado_icons/` にある SVG（`icon16.svg` 〜 `icon128.svg`）です。中身は4つとも同一で、`width` / `height` 属性だけが違います。

この環境には SVG をラスタライズするツール（`rsvg-convert` / `inkscape` / `cairosvg`）が入っていないため、同じ図形を Pillow で描き直して PNG を書き出しています。図形は「角丸背景 + 2×2 のタイル（左上が再生ボタン、右下が配信中マーク）」だけなので再現できます。

SVG のデザインを変えたときは、Pillow 側の座標も合わせて直す必要があります。頻繁に変えるなら `pip install cairosvg` で変換ツールを入れて、SVG から直接変換する方が確実です。

> **Warning:** `manifest.json` に `icons` を書いた状態で PNG が不在（または 0 バイト）だと、Chrome は「アイコンを読み込めませんでした」で**拡張機能そのものの読み込みに失敗**します。差し替えるときは4枚まとめて置いてください。
>
> 提出用の `package.sh` は、4枚が非空であることと `manifest.json` に `icons` があることの両方をチェックし、揃っていなければ ZIP を作らずに止まります。

なお、この README は `package.sh` が配布 ZIP から除外するため、ストアには含まれません。
